## What I Did

I contributed to the group analysis and final report.

To complete this project, I focused on analysing the problem, organising requirements, and preparing models or documentation that describe the proposed system. I reviewed the available files and outputs together so that the implementation, written explanation, and supporting evidence remained connected to the task objectives. Where the evidence did not clearly describe a personal contribution, I kept the description limited to the work that can be verified from this folder.

## Key Learning Outcomes

I learned how several analysis activities combine into one project document.

The work also strengthened my understanding of requirements analysis, process modelling, system design, documentation, and communication with stakeholders. I became more aware that a technical result should be supported by a clear process, suitable evidence, and an explanation of why the selected approach was appropriate. This will help me approach related tasks more systematically in future coursework.

## Reflection

The group project improved my ability to discuss system requirements and combine work from different members.

The most important challenge was keeping the requirements, diagrams, and proposed solution consistent with one another. Working through this required me to check the task in smaller stages and compare the result with the available requirements or evidence. This process improved my patience and made me more careful about verifying technical work before presenting it.

Overall, this project contributed to my development by combining practical work with documentation and reflection. It showed me which parts I can now complete with more confidence and which areas still require further practice. The experience will be useful when I work on larger projects that demand clearer organisation, stronger testing, and more independent technical decisions.

## Possible Improvements

A useful next step would be to add individual contribution, diagrams, and editable source files.

In a future version, I would also focus on a clearer traceability table, updated diagrams, and more explicit validation of the proposed design against user requirements. These additions would make the work easier to understand, reproduce, and evaluate while providing stronger evidence of the development process.
