## Reflection

The main challenge was keeping all diagrams and prototype pages consistent with the same requirements.

The most important challenge was keeping information from different project components consistent and connected to the same organisational problem. Working through this required me to check the task in smaller stages and compare the result with the available requirements or evidence. This process improved my patience and made me more careful about verifying technical work before presenting it.

Overall, this project contributed to my development by combining practical work with documentation and reflection. It showed me which parts I can now complete with more confidence and which areas still require further practice. The experience will be useful when I work on larger projects that demand clearer organisation, stronger testing, and more independent technical decisions.

## Possible Improvements

A useful next step would be to add captions and explain each diagram's purpose. Consolidate prototype pages into one navigable demonstration.

In a future version, I would also focus on stronger traceability between findings and designs, clearer diagram explanations, updated evidence, and a precise contribution statement. These additions would make the work easier to understand, reproduce, and evaluate while providing stronger evidence of the development process.
