# Project Data Collection

## Overview

This folder contains the questions used for project data collection.

## Objectives

- Gather information from relevant stakeholders or respondents.
- Support project requirements and analysis with collected evidence.

## Tools and Technologies

Questionnaire/interview documentation in Word and PDF formats.

## Folder Contents / Evidence

- [Data collection questions](<Data collection_Questions.pdf>)
- [Editable questions](Questions.docx)

## What I Did

I contributed to preparing the questions used to gather project information.

To complete this project, I focused on analysing the enterprise context and organising the available process, data, interface, and report evidence. I reviewed the available files and outputs together so that the implementation, written explanation, and supporting evidence remained connected to the task objectives. Where the evidence did not clearly describe a personal contribution, I kept the description limited to the work that can be verified from this folder.

## Key Learning Outcomes

I learned that useful requirements depend on clear, neutral, and relevant questions.

The work also strengthened my understanding of enterprise modelling, data collection, process analysis, system diagrams, interface design, and formal reporting. I became more aware that a technical result should be supported by a clear process, suitable evidence, and an explanation of why the selected approach was appropriate. This will help me approach related tasks more systematically in future coursework.

## Reflection

Designing questions required balancing detail with the time and knowledge of respondents.

The most important challenge was keeping information from different project components consistent and connected to the same organisational problem. Working through this required me to check the task in smaller stages and compare the result with the available requirements or evidence. This process improved my patience and made me more careful about verifying technical work before presenting it.

Overall, this project contributed to my development by combining practical work with documentation and reflection. It showed me which parts I can now complete with more confidence and which areas still require further practice. The experience will be useful when I work on larger projects that demand clearer organisation, stronger testing, and more independent technical decisions.

## Possible Improvements

A useful next step would be to add the collection method, respondent group, and summarised findings.

In a future version, I would also focus on stronger traceability between findings and designs, clearer diagram explanations, updated evidence, and a precise contribution statement. These additions would make the work easier to understand, reproduce, and evaluate while providing stronger evidence of the development process.
