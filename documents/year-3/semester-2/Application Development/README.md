- Flutter and Dart
- Local application storage

- Testing and project documentation

## Reflection

Application Development has allowed me to apply software-development knowledge through Flutter and Dart. The laboratory recorder application introduced interface components, local storage, and basic application behaviour, while the TVETMARA project expanded into Firebase, user authentication, Firestore, role-based functions, services, state, and testing. Together, the tasks demonstrate progression from a small application to a larger system.

The main challenge has been coordinating interface behaviour, application logic, stored data, and different user roles. A feature may appear correct visually while still producing an incorrect data update or permission flow. This has encouraged me to test behaviour from several perspectives and to organise the code so that responsibilities remain understandable as the project grows.

This subject has strengthened my confidence in building usable applications and connecting them to persistent data. It brings together lessons from object-oriented programming, databases, human-computer interaction, and software engineering. The experience is particularly valuable because it shows how data-engineering concepts can support applications used in practical organisational processes.
