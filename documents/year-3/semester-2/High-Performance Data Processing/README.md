## Reflection

High-Performance Data Processing has introduced methods for handling data more efficiently through memory awareness, alternative frameworks, concurrency, and distributed processing. The assignments and project involve pandas, Dask, Polars, asynchronous web crawling, performance measurement, and PySpark. The subject has encouraged me to evaluate how implementation choices affect execution time and resource use.

Creating a fair comparison is one of the main challenges. Performance can change because of data size, hardware, caching, framework behaviour, or differences in implementation. I have learned that optimisation should begin with measurement and that benchmark results need a clearly described environment and repeatable procedure before they can support a conclusion.

This subject has deepened my understanding of scalability, which is central to data engineering. It has shown me that a method that works for a small dataset may not remain suitable as volume or complexity increases. The experience is helping me make more informed choices about frameworks, concurrency, memory, and distributed processing in future data workflows.
