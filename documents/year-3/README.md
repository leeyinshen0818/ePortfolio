## 📝 Reflection

Year 3 connects my earlier programming, database, and analytical knowledge with larger data-oriented systems. Artificial Intelligence introduced more open-ended technical problems that required investigation and evaluation rather than a single fixed answer. Database Programming expanded my experience with relational SQL implementation and document-based data structures. Enterprise System Development & Modelling encouraged me to study processes, users, data, diagrams, and interfaces as connected parts of an organisational solution.

In Semester 2, the coursework became more closely related to practical data-engineering workflows. Application Development involves Flutter, Firebase, user roles, storage, and testing, while Business Intelligence focuses on transforming business records and communicating findings through dashboards. High-Performance Data Processing has introduced memory optimisation, framework comparison, asynchronous processing, web crawling, and PySpark. Special Topic in Data Engineering has provided exposure to Azure, Apache Spark, neural networks, DuckDB, dbt, and layered analytical pipelines.

The major challenge in Year 3 is managing systems with many components and selecting suitable tools for different purposes. I have learned that a working result is not enough; data quality, performance, maintainability, validation, usability, and documentation also need attention. Working with unfamiliar platforms has made me more comfortable reading technical documentation, testing alternatives, and learning independently. Overall, Year 3 has helped me understand how software development and data engineering support each other when creating practical products. It has also clarified the areas I want to continue improving, particularly scalable data processing, cloud platforms, reliable pipelines, and professionally structured applications.
