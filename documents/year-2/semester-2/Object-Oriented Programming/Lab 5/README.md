# School System Lab

## Overview

This lab contains a Java school-system program.

## Objectives

- Apply Java object-oriented design to a school-related domain.
- Practise class interaction and data management.

## Tools and Technologies

Java.

## Folder Contents / Evidence

- [SchoolSystem.java](SchoolSystem.java)

## What I Did

I implemented the assigned school-system functionality in Java.

To complete this laboratory task, I focused on implementing the required Java features through classes and clearly defined object responsibilities. I reviewed the available files and outputs together so that the implementation, written explanation, and supporting evidence remained connected to the task objectives. Where the evidence did not clearly describe a personal contribution, I kept the description limited to the work that can be verified from this folder.

## Key Learning Outcomes

I improved my ability to model real-world entities as program objects.

The work also strengthened my understanding of encapsulation, inheritance, polymorphism, collections, exception handling, and object-oriented design. I became more aware that a technical result should be supported by a clear process, suitable evidence, and an explanation of why the selected approach was appropriate. This will help me approach related tasks more systematically in future coursework.

## Reflection

This exercise helped me recognise when a program should be divided into smaller responsibilities. A single large source file is convenient initially but harder to maintain.

The most important challenge was keeping relationships between classes clear while avoiding duplicated responsibilities or unnecessary dependencies. Working through this required me to check the task in smaller stages and compare the result with the available requirements or evidence. This process improved my patience and made me more careful about verifying technical work before presenting it.

Overall, this laboratory task contributed to my development by combining practical work with documentation and reflection. It showed me which parts I can now complete with more confidence and which areas still require further practice. The experience will be useful when I work on larger projects that demand clearer organisation, stronger testing, and more independent technical decisions.

## Possible Improvements

A useful next step would be to split classes into separate files. A useful next step would be to add documentation, validation, and tests.

In a future version, I would also focus on class diagrams, more validation and exception cases, clearer execution instructions, and automated tests. These additions would make the work easier to understand, reproduce, and evaluate while providing stronger evidence of the development process.
