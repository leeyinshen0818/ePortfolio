## Reflection

Software Engineering introduced a disciplined process for planning, designing, testing, and documenting software. The coursework included requirements, design, test documentation, assignments, and a project overview. It taught me that software development is a coordinated process involving users, team members, documents, implementation decisions, and quality checks.

The main challenge was maintaining consistency between the different project documents. Requirements needed to be reflected in the design, and tests needed to demonstrate that the expected behaviour had been addressed. Teamwork also required clear communication and shared understanding so that separate contributions could form one coherent system.

This subject improved my ability to organise larger projects and explain technical decisions. I learned the value of traceability, version control, testing, and clearly defined responsibilities. These practices remain important in my later application and data-engineering work, where multiple components must be developed and verified without losing sight of the original objectives.
