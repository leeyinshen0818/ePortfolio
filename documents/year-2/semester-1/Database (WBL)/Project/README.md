# Database (WBL) Project

## Overview

This project contains database reports, an SRS, ERD images, and Draw.io modelling files for an application-approval-related system.

## Objectives

- Gather and document database requirements.
- Design entities, relationships, and application modules.
- Present the database design through formal documentation.

## Tools and Technologies

ERD, Draw.io, relational database concepts, and requirements documentation.

## Folder Contents / Evidence

- [Project report](Database_p2.pdf)
- [Software Requirements Specification](SRS.pdf)
- [ERD images](ERD)
- [Editable Draw.io models](DrawIO)

## What I Did

I contributed to the database requirements, ERD development, and project documentation.

To complete this project, I focused on designing, querying, and checking structured data using the available database evidence. I reviewed the available files and outputs together so that the implementation, written explanation, and supporting evidence remained connected to the task objectives. Where the evidence did not clearly describe a personal contribution, I kept the description limited to the work that can be verified from this folder.

## Key Learning Outcomes

I learned how requirements become entities, attributes, relationships, and database constraints.

The work also strengthened my understanding of relational modelling, SQL, table relationships, constraints, data integrity, and query construction. I became more aware that a technical result should be supported by a clear process, suitable evidence, and an explanation of why the selected approach was appropriate. This will help me approach related tasks more systematically in future coursework.

## Reflection

The project showed that an ERD must reflect the written requirements consistently. Revising relationships was a useful part of the design process.

The most important challenge was maintaining consistent relationships and writing queries that returned the intended result without introducing duplicate or invalid data. Working through this required me to check the task in smaller stages and compare the result with the available requirements or evidence. This process improved my patience and made me more careful about verifying technical work before presenting it.

Overall, this project contributed to my development by combining practical work with documentation and reflection. It showed me which parts I can now complete with more confidence and which areas still require further practice. The experience will be useful when I work on larger projects that demand clearer organisation, stronger testing, and more independent technical decisions.
